# PR #2 Remediation Plan — Iteration 001

## Findings to Fix

### Finding 1 (High) + Finding 2 (High): Remove `--user 10000:10000` and `runAsUser`/`runAsGroup`

**Problem:** `--user 10000:10000` in the `docker run` example and `runAsUser: 10000` / `runAsGroup: 10000` in the K8s Deployment force the container to start as UID 10000, breaking sshd privilege separation.

**User decision:** "We should not be requiring the user to add an explicit UID/GID at the commandline, we need to make a note in the README that this runs as user Hermes (10000:10000) so the user knows how to set proper permissions."

**Fix:**
- Remove `--user 10000:10000` from the `docker run` example in README.md
- Remove `runAsUser: 10000` and `runAsGroup: 10000` from the K8s Deployment `securityContext` in README.md
- Add a note in the `docker run` section explaining that SSH sessions run as `hermes` (UID 10000:10000) and the host-side `authorized_keys` file must be readable by the container (owned by UID 10000 or chmod 644)

### Finding 3 (Moderate): Replace `secrets: inherit` with explicit secret mapping

**Problem:** `secrets: inherit` passes all repository secrets to the external reusable workflow, not just the three required.

**User decision:** Valid finding. Needs remediation.

**Fix:**
- Replace `secrets: inherit` with explicit mapping of the three required secrets, matching the upstream shared-workflows README example:
  ```yaml
  secrets:
    DOCKERHUB_USERNAME: ${{ secrets.DOCKERHUB_USERNAME }}
    DOCKERHUB_TOKEN: ${{ secrets.DOCKERHUB_TOKEN }}
    DOCKERHUB_IMAGENAME: ${{ secrets.DOCKERHUB_IMAGENAME }}
  ```
